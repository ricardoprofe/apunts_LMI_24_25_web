1 Maquetació amb HTML i CSS

1.1 Reset i normalize

En treballar amb CSS els desenvolupadors sovint s’enfronten a inconsistències en la configuració del navegador, com ara les alçades de línia predeterminades, els marges i les mides de lletra dels encapçalaments. Aquestes inconsistències poden fer que el desenvolupament web siga difícil, ja que cada navegador aplica els seus propis estils predeterminats.

Per solucionar aquest problema i garantir un aspecte coherent a tots els navegadors, els desenvolupadors utilitzen dues tècniques principals: Reset CSS i Normalize CSS.

1.1.1 Reset

Aquesta tècnica consisteix a carregar un conjunt d’estils abans de qualsevol altre estil per eliminar els estils integrats que proporcionen els navegadors. Cada navegador té el seu propi full d’estil, que utilitza per fer que els llocs web sense estil siguen més llegibles. Per exemple, els navegadors solen mostrar els enllaços en blau i els enllaços visitats de color violeta de manera predeterminada, afegeixen vores i farciment a les taules i apliquen diferents mides de lletra als encapçalaments. Aquests estils predeterminats poden variar entre els navegadors, provocant inconsistències en la manera com es mostren les pàgines web.

Mitjançant CSS Reset, els desenvolupadors poden obligar a tots els navegadors a restablir els seus estils a null, minimitzant les diferències entre navegadors. Aquest enfocament garanteix que els desenvolupadors comencen amb una base coherent, lliure de les variacions en la representació predeterminada dels diferents navegadors.

Per exemple, un reset senzill seria:

* {
    margin: 0;
    padding: 0;
    border: 0;
    font-size: 100%;
}

Al següent enllaç tens un reset més complet:

https://meyerweb.com/eric/tools/css/reset/reset.css

El reset l’aplicariem abans dels nostres estils, per exemple:

<head>
  <link rel="stylesheet" href="reset.css">
  <link rel="stylesheet" href="general.css">
</head>

1.1.2 Normalize

Aquesta tècnica pretén fer que l’estil d’elements HTML siga coherent en diferents navegadors sense eliminar completament els estils predeterminats. Normalitzar CSS proporciona coherència entre navegadors alhora que preserva els valors predeterminats útils i s’adhereix als estàndards CSS moderns. En incloure un normalize al principi del fitxer CSS, ens assegurem que tots els navegadors comencen amb les mateixes regles d’estil.

Els objectius de la normalització són:

  • Conservar els estils útils predeterminats en lloc d’esborrar-los.
  • Estandaritzar estils per a una gran varietat d’elements HTML.
  • Corregir errors i incoherències dels navegadors habituals.
  • Millora la usabilitat mitjançant millores subtils.
  • Utilitza comentaris i documentació detallada per explicar el codi.

Al següent enllaç tens un exemple de normalize:

https://necolas.github.io/normalize.css/

A l’igual que el reset, l’hem d’aplicar abans que els nostres estils.

1.2 Maquetació amb Flexbox

El model de Caixes Flexibles (Flexbox) va ser dissenyat per a superar les dificultats del model de posicionament bàsic de CSS. Fa més fàcil crear un disseny flexible amb CSS.

Una caixa flexbox sempre consta de:

  • Un contenidor flex: l’element contenidor pare, que pot ser <div>, <nav>, etc.
  • Elements flexibles: els elements dins del contenidor.

Per a definir un contenidor flex, simplement li hem d’afegir la propietat display: flex:

...
<style>
  nav#menu ul {
      display: flex;
      background-color: lightslategray;
      list-style: none;
  }

  nav#menu li {
      border: 1px solid blue;
      border-radius: 5px;
      background-color: azure;
      padding: 5px;
      text-align: center;
  }

  nav#menu li a{
      text-decoration: none;
      text-align: center;
  }
</style>
...
<nav id="menu">
  <ul>
      <li><a href="#">Link 1</a></li>
      <li><a href="#">Link 2</a></li>
      <li><a href="#">Link 3</a></li>
      <li><a href="#">Link 4</a></li>
  </ul>
</nav>

Podem indicar la direcció del contenidr amb la propietat flex-direction, la qual pot tindre un dels següents valors:

  • row: en fila, valor per defecte.
  • column: en columna
  • row-reverse: en fila en ordre invers.
  • column-reverse: en columna en ordre invers.

Per exemple:

nav#menu ul {
  display: flex;
  flex-direction: column;
  ...
}

justify-content alinea els elements horitzontalment. Els valors més importants són center, flex-start i flex-end.

nav#menu ul {
  display: flex;
  justify-content: flex-end;
  ...
}

align-items s’utilitza per alinear els elements verticalment quan no utilitzen tot l’espai disponible a l’eix vertical, per exemple, quan tenim un height definit en píxels al contenidor.

Els valor més usats són center, flex-start, flex-end.

nav#menu ul {
  display: flex;
  height: 100px;
  align-items: center;
  ...
}

Amb justify-content i align-items podem centrar de forma molt fàcil un element en un contenidor:

.flex-container {
  display: flex;
  height: 300px;
  justify-content: center;
  align-items: center;
}

Podem usar la propietat gap per a definir la separació entre elements:

nav#menu ul {
  display: flex;
  gap: 10px;
  ...
}

Quant als elements dins d’un contenidor flexbox, podem establir 3 propietats:

  • flex-grow especifica quant creix un element en relació amb la resta d’elements.
  • flex-shrink especifica quant es reduirà un element en relació amb la resta d’elements.
  • flex-basis especifica la longitud inicial d’un element flexible.
  • flex és una propietat abreviada per a les propietats flex-grow, flex-shrink i flex-basis.

flex-grow i flex-shrink s’indiquem amb nombres enters (ex. un element amb flex-grow: 2 ocuparà el doble que la resta d’elements), mentres que flex-basis s’indica amb unitats de mesura, com píxels.

Per exemple:

.flexitem {
    flex: 1 2 200px;
}

és el mateix que:

.flex-item {
    flex-grow: 1;
    flex-shrink: 2;
    flex-basis: 200px;
}

Si indiquem només una propietat amb flex, aquestà s’aplicarà a flex-grow, deixant les altres al seu valor per defecte:

.flexitem {
    flex: 2;
}

és el mateix que:

.flex-item {
    flex-grow: 2;
    flex-shrink: 1;
    flex-basis: auto;
}

1.2.1.1 Menú amb Flexbox

flexbox simplifica el procés de creació d’un menú:

<style>        
  nav#menu ul {
      display: flex;
      align-items: flex-start;
      gap: 10px;
      padding: 10px;
      background-color: lightslategray;           
      list-style: none;
  }

  nav#menu li {
      border: 1px solid blue;
      border-radius: 5px;
      background-color: azure;
      padding: 5px;
      text-align: center;
  }

  nav#menu li a{
      text-decoration: none;
      text-align: center;
  }

  .push-right {
      margin-left: auto;
  }
</style>
...
<nav id="menu">
  <ul>
    <li><a href="#">Link 1</a></li>
    <li><a href="#">Link 2</a></li>
    <li><a href="#">Link 3</a></li>
    <li><a href="#">Link 4</a></li>

</nav>

Podem fer inputs de formularis amb una aparença més professional:

<style>        
  .wrapper {
      display: flex;
      width: 400px;
      border: 1px solid blue;
      border-radius: 5px;
  }

  .wrapper input[type="text"] {
      flex: 1 1 auto;
      background-color: lightblue;
      border: none;
  }

  .wrapper label {
      background-color: dimgrey;
      color: white;
      padding: 5px;
  }

  .wrapper input[type="submit"] {
      background-color:dodgerblue;
      color: white;
      border: none;
      padding: 5px;
  }        
</style>
...
<form>
    <div class="wrapper">
        <label for="text">Label</label>
        <input type="text" id="text" placeholder="Write here">
        <input type="submit" value="Submit">
    </div>
</form>

1.3 Maquetació amb Grid

El mòdul CSS Grid Layout ofereix un sistema de disseny basat en una quadrícula composta per files i columnes. Amb aquesta eina, els desenvolupadors poden crear dissenys web complexos de manera senzilla i estructurada.

A diferència d’altres mòduls de disseny, CSS Grid permet dissenyar estructures responsives sense necessitat d’utilitzar flotacions (float) o posicionaments manuals (position). Gràcies a les seues propietats es pot definir amb precisió l’espai i la col·locació dels elements dins de la quadrícula.

Un altre avantatge de CSS Grid Layout és la seua compatibilitat amb tots els navegadors moderns, la qual cosa garanteix que el disseny es visualitzarà correctament en qualsevol dispositiu i plataforma.

1.3.1 Grid vs. Flexbox

Quan es tracta d’escollir entre CSS Grid i Flexbox, és important tenir en compte l’estructura del disseny que volem aconseguir:

  • Grid s’ha d’utilitzar per a dissenys bidimensionals, on cal treballar tant amb files com amb columnes. Això el fa ideal per a dissenys de pàgines web senceres o seccions complexes amb diferents dimensions.

  • Flexbox s’ha d’utilitzar per a dissenys unidimensionals, on es treballa amb una sola direcció, ja siga en fila o en columna. És més adequat per a alinear elements dins d’un contenidor o per a estructures més simples.

1.3.2 Plantilla amb Grid

Grid sempre consta de 2 tipus d’elements:

  • Grid Container: l’element pare contenidor.
  • Elements del grid: els elements dins del contenidor.

Per a que un contenidor siga grid container, hem d’usar la propietat display: grid:

.container {
  display: grid;
  ...

Un bon exemple d’us de grid, és la maquetació d’una plantilla web com la de la imatge:

Per a començar definim un contenidor i la distribució dels elements amb grid-template-areas:

.container {
  display: grid;
  grid-template-areas:
      "header header"
      "menu content"
      "footer footer";
      ...

Ací estem definint 3 files i 2 columnes, de manera que els elements de la primera i última fila, header i footer, ocupen 2 columnes, mentres que menu i content ocupen una columna només.

Afegim al contenidor la propietat grid-template-columns per definir l’ample de cada columna

grid-template-columns: 100px auto;

Amb aquest codi estem definint que la primera columna ocupe 100px d’ample i la segona la resta de l’espai.

Per últim, amb gap definim la separació entre elements:

gap: 5px;

Quant als elements, utilitzem la propietat grid-area per a donar un nom a cada element i relacionar-los amb les noms de la propietat grid-template-areas del contenidor:

header {
    grid-area: header;
    text-align: center;
}

nav#menu {
    grid-area: menu;
}

main {
    grid-area: content;
}

footer {
    grid-area: footer;
}

El codi complet:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Exemple maquetacio grid</title>
    <style>
        body {
            margin: 0;
            padding: 0;
        }

        main, footer, header, nav {
            padding: 10px;
            background-color: aliceblue;
        }

        .container {
            display: grid;
            grid-template-areas:
                "header header"
                "menu content"
                "footer footer";
            grid-template-columns: 100px auto;
            gap: 5px;
            background-color: #2196F3;
            padding: 5px;
        }

        header {
            grid-area: header;
            text-align: center;
        }

        nav#menu {
            grid-area: menu;
        }

        main {
            grid-area: content;
        }

        footer {
            grid-area: footer;
        }
    </style>
</head>

<body>
    <div class="container">
        <header>
            <p>{header}</p>
            <h1>My Header</h1>
        </header>
        <nav id="menu">
            <p>{menu}</p>
            <a href="#">Link 1</a><br>
            <a href="#">Link 2</a><br>
            <a href="#">Link 3</a>
        </nav>
        <main>
            <p>{main}</p>
            <h2>Lorem Ipsum</h2>
            <p>Lorem ipsum odor amet, consectetuer adipiscing elit. Ridiculus sit nisl laoreet facilisis aliquet. Potenti dignissim litora eget montes rhoncus sapien neque urna. Cursus libero sapien integer magnis ligula lobortis quam ut.</p>
        </main>
        <footer>
            <p>{footer}</p>
        </footer>
    </div>
</body>

</html>

1.3.3 Galeria amb grid

Una altra opció senzilla és crear una galeria o una disposició en graella per a mostrar targetes.

Primer definim el contenidor al qual li assignem en nom de classe gallery:

.gallery {
    display: grid;
    grid-template-columns: auto auto auto; 
    gap: 15px;
}

Amb grid-template-columns estem definint 3 columnes amb la mateixa amplària (auto auto auto).

Si volguerem un ample fixe, també ho podem fer amb:

grid-template-columns: 300px 300px 300px;

El CSS per a les imatges:

.gallery img {
    width: 100%;
    height: auto;
}

Amb això ajustem la mida de les imatges a l’ample de les columnes.

Si tinguerem 7 imatges a la galeria, el resultat seria:

Si volem que un element del grid ocupe més d’una cel·la de la graella, li assignem una classe i usem la propietat grid-column:

.gallery_item1 {
    grid-column: 1 / span 3;
}

Amb aquest codi estem especificant que l’element comence en la columna 1 i s’expandisca 3 columnes:

La propietat grid-row és similar però funciona amb files. Es poden combinar ambdues propietats:

.gallery_item1 {
    grid-column: 1 / span 2;
    grid-row: 1 / span 2;
}

El codi complet:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Exemple maquetacio grid</title>
    <style>    
        .gallery {
            display: grid;
            grid-template-columns: auto auto auto; 
            gap: 15px;
            background-color: #2196F3;
            padding: 15px;
            width: 80%;
            margin: auto;
        }

        .gallery img {
            width: 100%;
            height: auto;
        }        
        .gallery_item1 {
            grid-column: 1 / span 2;
            grid-row: 1 / span 2;
        }
    </style>
</head>

<body>
    <div class="gallery">
        <img class="gallery_item1" src="ies-jaume-ii-el-just.jpg" alt="ies">
        <img src="ies-jaume-ii-el-just.jpg" alt="ies">
        <img src="ies-jaume-ii-el-just.jpg" alt="ies">
        <img src="ies-jaume-ii-el-just.jpg" alt="ies">
        <img src="ies-jaume-ii-el-just.jpg" alt="ies">
        <img src="ies-jaume-ii-el-just.jpg" alt="ies">
        <img src="ies-jaume-ii-el-just.jpg" alt="ies">
    </div>
</body>

</html>

1.4 Maquetació amb position

La maquetació amb la propietat position de CSS s’utilitzava àmplament abans de l’aparició dels model Flexbox i Grid. Encara es pot utilitzar

position permet modificar el posicionament amb el que es mostra cada caixa.

El posicionament en CSS es basa en quatre opcions:

  • Posicionament Estàtic o Normal
  • Posicionament Relatiu
  • Posicionament Flotant
  • Posicionament Absolut

S’utilitza la propietat position.

position: static | relative | absolute | fixed

1.4.1 Posicionament Normal (static)

Es tracta del funcionament per defecte.

Les caixes apareixen una darrere l’altra i de dalt a baix.

Només es té en compte si l’element és en bloc o en línia.

div.static {
  position: static;
}

1.4.2 Posicionament Relatiu (relative)

Consisteix en posicionar la caixa segons el posicionament normal i després desplaçar-la respecte de la seva posició original.

  • Canviar la posició relativa pot fer que el contingut de dues caixes quede superposat.

  • Es marca la posició original de l’element com a protegida (la resta de caixes es pensen que encara hi és).

El desplaçament de la caixa es controla amb les propietats top, right, bottom i left.

top, right, bottom i left es calculen respecte de la posició original de la caixa.

div.relative {
  position: relative;
  top: 25px;
  right: 25px;
}

  • top:25px: es desplaça 25 píxels des de dalt de la posició normal de la caixa (es desplaça cap a baix).

  • right:25px: es desplaça 25 píxels de la dreta de la posició original (es desplaça cap a l’esquerra).

1.4.3 Posicionament fixed

Ens fixa una caixa en la pantalla de manera que no es mourà encara que es moga la pàgina amunt o avall.

.box1{ 
    position:fixed; 
    right:0px; 
    top:0px; 
}

1.4.4 Posicionament Absolut (absolute)

Un element amb position:absolute es posiciona en relació amb l’ancestre posicionat més proper.

Si un element posicionat absolut no té ancestres posicionats, utilitza el body del document i es mou amb el desplaçament de la pàgina.

Nota: els elements amb posicionament absolut s’eliminen del flux normal i es poden solapar amb altres elements.

<style>
div.relative {
  position: relative;
  width: 400px;
  height: 200px;
  border: 3px solid #73AD21;
} 

div.absolute {
  position: absolute;
  top: 80px;
  right: 0;
  width: 200px;
  height: 100px;
  border: 3px solid #73AD21;
}
</style>
...
<div class="relative">Aquest element té position: relative;
  <div class="absolute">Aquest element té position: absolute;</div>
</div>

Exemple.

1.4.5 Posicionament flotant (float)

Podem definir una caixa flotant que deixa que les altres caixes es posin al seu voltant amb la propietat float. Els valors possibles són left, right o none.

#caja1 {
  float: right;
}

La caixa que hem definit amb float:right es posa primer, al més a la dreta o esquerra possible, i les altres es posen al seu voltant sense sobreescriure-la.

La propietat clear permet anul·lar el comportament introduït per la propietat float.

clear: left anul·la els elements flotants a l’esquerra.

La propietat clear pot tenir un dels valors següents:

  • left: l’element es mou per baix dels elements flotants esquerre
  • right: l’element es mou per baix dels elements flotants a la dreta
  • both: l’element es mou per baix dels elements flotants esquerre i dret
  • none: l’element no es mou per baix dels elements flotants esquerre o dret. Valor per defecte.

Exemples de float.

1.4.6 Posicionament sticky (sticky)

Un element amb position: sticky es posiciona en funció de la posició de desplaçament (scroll) de l’usuari.

Un element sticky alterna entre relatiu i fix, depenent de la posició de desplaçament. Es col·loca relativament fins que es troba una posició de desplaçament determinada a la finestra gràfica; després s’enganxa al seu lloc (com la posició fixa).

S’ha d’especificar una posició i un valor amb top, bottom, left o right. Exemple:

div.sticky {
  position: sticky;
  top: 0;
}

Exemple de posicionament sticky

1.4.7 Z-index

la propietat z-index controla la profunditat de les capes. Quan més alt siga el seu valor, més a prop de l’usuari es veurà l’element:

img {
  position: absolute;
  z-index: -1;
}
